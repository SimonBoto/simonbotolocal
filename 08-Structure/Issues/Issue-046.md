## Implementation Plan

### Phase 1: Planning Engine (2 hours)
1. Create `night-plan-generator.sh`
   - Reads active LitNotes from 01-Literature/
   - Prioritizes by: GOLD tier, age, zettel count
   - Generates batch of 10-15 LitNotes
   - Writes plan to `pending/night-plan-YYYY-MM-DD.json`

2. Create validation criteria:
   - YAML must parse (yq check)
   - All links must resolve (link-check script)
   - Minimum 3 zettels per LitNote
   - Maximum 20% failure rate per batch

### Phase 2: Execution Engine (2 hours)
1. Create `night-executor.sh`
   - Reads plan from pending/
   - Spawns FORG with batch list
   - Monitors progress via checkpoint files
   - Auto-retries failed items (max 2 attempts)
   - Git commits each completed LitNote

2. Create `morning-report-generator.sh`
   - Parses git log for night's commits
   - Counts: LitNotes processed, zettels created, errors
   - Flags anomalies for human review
   - Writes report to `08-Structure/Daily/YYYY-MM-DD-night-report.md`

### Phase 3: Integration (1 hour)
1. Cron configuration:
   ```
   30 22 * * * /home/simon/.openclaw/workspace/night-shift/night-plan-generator.sh
   35 22 * * * /home/simon/.openclaw/workspace/night-shift/night-executor.sh
   30 06 * * * /home/simon/.openclaw/workspace/night-shift/morning-report-generator.sh
   ```

2. K2.6 integration:
   - Reviews morning report
   - Decides: auto-fix vs. flag for human
   - Updates ISSUE status
   - Adjusts next night's plan based on errors

### Phase 4: Testing (1 hour)
1. Dry run: simulate night shift without actual processing
2. Pilot: 1 night with 5 LitNotes
3. Scale: 7 consecutive nights
4. Document: create Skill-Night-Shift_v1.0.md
