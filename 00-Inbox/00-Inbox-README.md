# 00-Inbox — README

> **Capture everything. Process daily. Empty regularly.**

---

## Purpose

The Inbox is a **capture zone** — not storage, not organization, just *collection*.

Everything enters here first:
- CGPT conversations
- Web articles
- Book notes
- Ideas
- Tasks

**Rule:** Inbox is temporary. Process within 24-48 hours.

---

## Inbox Zero Protocol

**Daily (or every session):**
1. **Review** — Scan all items
2. **Decide** — Each item gets one action:
   - **Process** → Move to 01-Literature/ or 03-Zettels/
   - **Defer** → Move to Future/
   - **Delete** → Remove if useless
3. **Empty** — Inbox should be clear

**Never:** Leave items in Inbox for >48 hours.

## Fleeting vs Future

| Folder | Use For | Review |
|--------|---------|--------|
| **Fleeting/** | Quick captures, may become zettels | Weekly |
| **Future/** | Deferred items with specific trigger | Monthly |

---

## Rules

1. **Capture liberally** — Don't filter at input
2. **Process ruthlessly** — Most things get deleted
3. **Empty regularly** — Inbox Zero is the goal
4. **No organization** — That's for later stages
