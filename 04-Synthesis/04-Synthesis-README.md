# 04-Synthesis — README

> **Higher-order knowledge — integration of zettels into living documents.**

---

## Purpose

Synthesis is **integration** — combining atomic zettels into coherent, useful documents.

Unlike zettels (atomic, permanent), synthesis is:
- **Living** — Evolves as knowledge grows
- **Integrated** — Combines multiple sources
- **Applied** — Solves specific problems
- **Narrative** — Tells a coherent story

---

## Types of Synthesis

| Type | Example | Use Case |
|------|---------|----------|
| **Protocol** | Clinical protocol | Step-by-step procedure |
| **Review** | Adipic acid review | Comprehensive overview |
| **Analysis** | METHAP study design | Research planning |
| **Guide** | Weekly consolidation ritual | Process documentation |
| **Framework** | Decision frameworks | Thinking tools |

## Synthesis vs Zettels

| Aspect | Zettels | Synthesis |
|--------|---------|-----------|
| **Granularity** | Atomic (one idea) | Integrated (many ideas) |
| **Stability** | Permanent (Subconscious/) | Living (evolves) |
| **Purpose** | Reference | Application |
| **Links** | Many lateral connections | Narrative flow |

**Relationship:** Zettels feed synthesis. Synthesis reveals gaps → new zettels.

## When to Create Synthesis

1. **Critical mass** — 5+ related zettels on same topic
2. **Application need** — Clinical protocol, research design
3. **Pattern emergence** — Connections become clear
4. **Communication** — Need to explain to others

---

## Rules

1. **Synthesis cites zettels** — Grounded in atomic knowledge
2. **Living documents evolve** — Update as knowledge grows
3. **Archive old versions** — Don't delete, preserve history
4. **Link back to sources** — Maintain provenance
