## 🔄 OPTIMAL FLOWCHARTS BY ROLE

### CEO (Dr. Simos) — Flowchart

```
START: New idea / Task / Priority shift
              │
              ▼
    Write 00-COMMAND.md
    ├─ P0: Must do (critical path)
    ├─ P1: Should do (progress)
    ├─ P2: Backlog (when time)
    └─ Standing Orders (always true)
              │
              ▼
    Wait or Continue own work
              │
              ▼
    Check 01-QUEUE.md (what RON doing)
              │
              ▼
    Review 02-HANDOFF.md (results)
              │
    ├─ Approve all → Say "Approve all"
    ├─ Review X → Say "Review X" 
    ├─ Redo Y → Say "Redo Y with [changes]"
    └─ Edit directly → Modify HANDOFF.md
              │
              ▼
    If RED risk needed → Say "I ACCEPT RED"
              │
              ▼
    Repeat or End
```

**CEO Time Allocation:**
- 60% — Clinical work, research, reading
- 20% — Reviewing HANDOFF, giving feedback
- 15% — Writing COMMAND, setting priorities
- 5% — Direct RON interaction (questions, clarifications)

### AI Boss (RON) — Flowchart

```
START: Session begins or message received
              │
              ▼
    Read 00-COMMAND.md
    + Recent 03-LOG.md for context
              │
              ▼
    Plan execution strategy
    ├─ Brain work? → Do myself (kimi-k2.5)
    ├─ Batch/Parallel? → Spawn GROQ
    ├─ Routine/Test? → Spawn FREE
    └─ Quality/Critical? → Spawn PAID
              │
              ▼
    Update 01-QUEUE.md
    "Currently processing..."
              │
              ▼
    EXECUTE
    ├─ Monitor agent health
    ├─ Handle failures/retry
    └─ Collect results
              │
              ▼
    Synthesize outputs
    ├─ Write 02-HANDOFF.md (for review)
    ├─ Update 03-LOG.md (record)
    └─ Update 00-LIVE-STATUS.md (current)
              │
              ▼
    If YELLOW operation → Report immediately
    If completed → Await CEO review
              │
              ▼
    Check for new COMMAND
    or Heartbeat (every 4 hours)
              │
              ▼
    REPEAT
```

**RON Time Allocation:**
- 30% — Reading COMMAND, planning
- 40% — Executing (direct work + managing agents)
- 20% — Writing QUEUE, HANDOFF, LOG
- 10% — Maintenance, learning, improvement

### GROQ — Flowchart

```
START: Spawned by RON for batch task
              │
              ▼
    Receive task + context
              │
              ▼
    Check rate limits
    ├─ 30 RPM available? → Proceed
    └─ Near limit? → Backoff, report
              │
              ▼
    EXECUTE (llama-3.3-70b)
    ├─ Parallel where possible
    ├─ Fast inference priority
    └─ Complete batch
              │
              ▼
    Return results to RON
              │
              ▼
    DIE (session ends)
```

**GROQ Constraints:**
- 30 requests per minute max
- 200K tokens per minute max
- Best for: Batch extraction, parallel processing
- Avoid: Complex reasoning, long context

### PAID — Flowchart

```

              ▼
```

**PAID Constraints:**
- $2/day hard cap

- Escalate to GPT-4o/Claude only when justified
