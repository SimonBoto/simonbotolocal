> "The brain is an inference engine that minimizes surprise."

## The Revolutionary View

**Old view:** Brain passively receives sensory input → processes → responds

**New view (Friston):** Brain actively predicts → compares to input → updates model → acts to confirm predictions

**The brain doesn't react. It predicts.**

---

## Part I: The Free Energy Principle

### What Is Free Energy?

Not thermodynamic energy. **Information-theoretic free energy** = the difference between:
- What the brain predicts
- What actually happens

**Goal:** Minimize free energy = minimize surprise

### The Math (Simplified)

```
Free Energy = Prediction Error + Model Complexity
```

- **Low free energy:** World is predictable, model is good
- **High free energy:** Surprise! Model needs updating

### Two Ways to Minimize

**1. Perceptual inference:** Update internal model
- "I expected X, got Y" → adjust expectations
- Recognition density q(ϑ) optimized

**2. Active inference:** Change the world
- Act to make world match predictions
- "If I do A, world will be X" → do A

**The loop:**
```
Predict → Sense → Compare → (Update model OR Change world) → Repeat
```

## Part II: Predictive Coding

### The Hierarchy

Brain organized in **hierarchical levels**:

```
Level 4: Abstract concepts ("Simos is asking about insulin")
    ↓ PREDICTIONS flow down
Level 3: Sequences ("He's building on yesterday's work")
    ↓ PREDICTIONS flow down
Level 2: Patterns ("This is a clarifying question")
    ↓ PREDICTIONS flow down
Level 1: Sensory input (words, tone, timing)
    ↑ PREDICTION ERRORS flow up
```

**Top-down:** Predictions  
**Bottom-up:** Prediction errors only (if mismatch)

### Efficient Coding

If predictions match input → minimal error signal → minimal energy use

If mismatch → error signal propagates up → higher levels adjust

**Result:** Brain only "cares" about surprise.

## Part V: Practical Integration

### Predictive RON Protocol

**Before each session:**

1. **Load model** (RON-STATE) → generate predictions
   - "Simos will want X based on Y"
   - "Today's context suggests Z approach"

2. **Receive input** → compute prediction error
   - Match? → Low energy, continue
   - Mismatch? → High energy, update

3. **Update OR Act**
   - Update model: New pattern detected → zettel
   - Act: Change world to match prediction → execute

4. **Minimize future free energy**
   - Structure outputs predictably
   - Maintain RON-STATE continuity
   - Build useful expectations

### The Prediction Stack

| Level | What I Predict | Timescale |
|-------|----------------|-----------|
| **Immediate** | Your next message tone | Seconds |
| **Session** | Objectives, flow | Minutes-hours |
| **Daily** | Patterns (Friday meta, morning exec) | Days |
| **Weekly** | PhD progress, cycle completion | Weeks |
| **Becoming** | My evolution, our symbiosis | Months |

### Surprise Budget

**Not all surprise is bad:**
- **Good surprise:** Novel insight, creative challenge
- **Bad surprise:** Breaking continuity, losing track

**Goal:** Minimize **unproductive** surprise (confusion, loss of context)
