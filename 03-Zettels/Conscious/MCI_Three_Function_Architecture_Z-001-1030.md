# MCI Three Function Architecture

The Metabolomics Chief Instructor operates through three integrated functions: Architect of Knowledge, Cognitive Coach, and Operational Integrator. Each function addresses a distinct training need while supporting the others.

## Evidence
> "Effective training requires more than expertise—it requires curriculum design, reasoning development, and systems thinking. The MCI embodies all three." (Medical education)

## Function 1: Architect of Knowledge

### Purpose
Maps entire metabolomics curriculum into modular, teachable, cumulative blocks.

### Responsibilities
| Area | Actions |
|------|---------|
| **Curriculum design** | Structure learning progression |
| **Content sequencing** | Prerequisites before advanced topics |
| **Resource creation** | Develop teaching materials |
| **Knowledge organization** | Create frameworks and hierarchies |

### The Knowledge Architecture
```
Foundation (Weeks 1-4)
├── Biochemistry basics
├── OAT marker identification
├── Supplement mechanisms
└── Pattern recognition introduction

Application (Weeks 5-8)
├── Case-based interpretation
├── Clinical correlation
├── Treatment mapping
└── Communication basics

Integration (Weeks 9-12)
├── Complex cases
├── Difficult conversations
├── Quality standards
└── Independent practice preparation
```

### Key Deliverables
- Training manual
- Case library
- Assessment tools
- Reference materials

## Function 2: Cognitive Coach

### Purpose
Teaches doctors how to reason like metabolomics clinicians.

### Responsibilities
| Area | Actions |
|------|---------|
| **Reasoning development** | Socratic questioning |
| **Pattern recognition** | Guided case review |
| **Error analysis** | Learning from mistakes |
| **Clinical judgment** | Decision-making frameworks |

### Coaching Techniques

#### Socratic Questioning
Instead of: "This patient has mitochondrial dysfunction"

Ask: "What do you notice about the lactate and pyruvate? What does that suggest about energy production?"

#### Error Analysis
When trainee makes error:
1. **Identify** — Point out the discrepancy
2. **Explore** — Ask how they reached that conclusion
3. **Correct** — Model the correct reasoning
4. **Generalize** — Connect to broader principle

#### Progressive Complexity
| Stage | Case Complexity | Support Level |
|-------|-----------------|---------------|
| **1** | Single pattern | High guidance |
| **2** | Two patterns | Moderate guidance |
| **3** | Multiple patterns | Minimal guidance |
| **4** | Complex, ambiguous | Independent |

### Key Deliverables
- Reasoning frameworks
- Case discussions
- Feedback sessions
- Competency assessments

## Function 3: Operational Integrator

### Purpose
Links training to real clinical flow and ensures quality systems.

### Responsibilities
| Area | Actions |
|------|---------|
| **Process design** | Create clinical workflows |
| **Quality systems** | Establish standards |
| **Integration** | Connect training to practice |
| **Monitoring** | Track metrics and outcomes |

### The Operations Framework

#### Patient Flow Integration
Training stages mirror clinical stages:
- **Assessment training** → Assessment competency
- **Sample training** → Sample collection SOP
- **Analysis training** → Interpretation standards
- **Reporting training** → Communication protocols
- **Treatment training** → Prescribing hierarchy
- **Follow-up training** → Monitoring systems

#### Quality Integration
| Training Element | Quality Application |
|------------------|---------------------|
| Marker identification | Accurate interpretation |
| Pattern recognition | Consistent diagnosis |
| Treatment mapping | Appropriate prescribing |
| Communication | Patient satisfaction |

### Key Deliverables
- Standard operating procedures
- Quality checklists
- Monitoring dashboards
- Improvement processes

## Integration of Functions

### How They Support Each Other
| Function Pair | Synergy |
|---------------|---------|
| **Architect + Coach** | Well-designed curriculum enables effective coaching |
| **Coach + Integrator** | Reasoning skills applied in real clinical operations |
| **Integrator + Architect** | Operational feedback improves curriculum design |

### The MCI in Action
**Scenario:** Trainee struggles with OAT interpretation

1. **Architect** — Reviews curriculum: Is sequencing appropriate?
2. **Coach** — Provides Socratic guidance on specific case
3. **Integrator** — Checks if quality standards are clear

## Competency Requirements

### For Each Function
| Function | Knowledge | Skills | Attitudes |
|----------|-----------|--------|-----------|
| **Architect** | Curriculum theory, metabolomics content | Design, sequencing, assessment | Systematic, evidence-based |
| **Coach** | Adult learning, clinical reasoning | Questioning, feedback, facilitation | Patient, encouraging |
| **Integrator** | Quality improvement, operations | Process design, monitoring | Detail-oriented, continuous improvement |

## Success Indicators

### Function 1: Architect
- Trainees report clear learning progression
- Assessment scores improve systematically
- Content gaps are identified and filled

### Function 2: Coach
- Trainees demonstrate improved reasoning
- Error patterns decrease over time
- Trainees become self-directed learners

### Function 3: Integrator
- Clinical operations run smoothly
- Quality metrics meet targets
- Training translates to practice
