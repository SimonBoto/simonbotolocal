# 00-Inbox — README

> **Capture everything. Process ruthlessly. Empty daily.**

---

## Purpose

The Inbox is a **temporary capture zone** — nothing lives here permanently.

Every input enters here first:
- Conversations (Grok 4.1, Kimi K2.6, Nemotron, etc.)
- Articles, PDFs, web clips
- Book notes, voice memos
- Raw ideas, tasks, insights

**Hard Rule**: Nothing stays longer than 24–48 hours.

---

## Folder Structure

- **Fleeting/** → Raw, unprocessed captures
- **Processed/** → RON-reviewed & extracted
- `00-Inbox-README.md`
- `00-Inbox-Index.md`

---

## Inbox Zero Protocol (RON-owned)

**Every session / daily RON task:**

1. **Review** all Fleeting items
2. **Decide** for each item:
   - **Process** → Extract atomic Zettels / LitNotes → move source to Processed/
   - **Defer** → move to Future/ (if you keep the folder) or delete
   - **Delete** → ruthlessly
3. **Empty** Fleeting/ completely

**FORG** may only drop raw material into Fleeting/.  
**RON** owns all processing and promotion.

---

## Rules (Aligned with 00-MISSION)

1. Capture liberally, process ruthlessly
2. One idea per Zettel
3. Every processed note **must** receive UID + YAML frontmatter
4. Always run QMD semantic search before creating anything new
5. Inbox Zero = system integrity
